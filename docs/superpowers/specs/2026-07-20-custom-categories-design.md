# 사용자 정의 카테고리 설계

날짜: 2026-07-20 · 상태: 사용자 승인 완료

## 목적

지출/수입 카테고리가 `app.js`에 하드코딩되어 있어 사용자가 자기 소비 패턴에 맞는
카테고리를 쓸 수 없다. 사용자가 직접 카테고리를 추가하고, 자신이 만든 카테고리에
한해 이름 변경·삭제할 수 있게 한다. 기본 카테고리는 고정 유지한다.

## 저장 구조

- 상태에 `customCategories: { expense: [], income: [] }` 추가 (문자열 배열).
- `defaultState()`, `loadState()` 병합, v2 백업 복원 경로에 포함.
  localStorage(`one_hundred_million_mobile_v2`)와 백업 파일에는 전체 상태가
  저장되므로 자동 포함된다.
- 칩 표시 목록 = 기본 카테고리 + 커스텀 카테고리 (기본이 앞).

## 추가 UX (지출/수입 입력 시트)

- 카테고리 칩 줄 맨 끝에 `+ 추가` 칩 (대시 보더로 액션임을 구분).
- 탭 → `window.prompt`로 이름 입력 (기존 계좌 추가와 동일 패턴).
- 검증: 공백 정리 후 빈값·8자 초과·중복(기본+커스텀, 양 타입 무관 동일 타입 내
  기준) 거부. 거부 시 `window.alert`로 사유 안내.
- 성공 시 즉시 저장하고 칩 줄을 다시 그려 새 카테고리를 선택 상태로 만든다.
- 지출·수입 시트 모두 지원.

## 관리 UX (설정 탭)

- 설정 탭에 "카테고리" 섹션 추가. **내가 만든 카테고리만** 행으로 나열
  (지출/수입 구분 표기). 비어 있으면 "기록 입력에서 + 추가로 만듭니다" 힌트만.
- 행 탭 → 액션 시트 (계좌 관리와 동일 패턴): **이름 변경 / 삭제**.
- 이름 변경: 해당 카테고리를 쓰던 과거 거래(`transactions`)와 고정
  항목(`fixedItems`)의 `category`도 함께 새 이름으로 이관한다. 흐름 탭 집계가
  두 이름으로 갈라지지 않게 하기 위함. (사용자 확인: 과거 기록도 바뀌는 동작 승인)
- 삭제: `window.confirm` 후 칩만 제거. 과거 기록의 카테고리 문자열은 유지되어
  계속 표시·집계된다.

## 순수 로직 (logic.js + tests.js)

- `validateNewCategory(rawName, existingNames)` — 공백 정리, 빈값/8자 초과/중복이면
  `null`, 아니면 정규화된 이름 반환.
- `renameCategoryInRecords(transactions, fixedItems, from, to)` — category가 `from`인
  항목을 `to`로 바꾼 새 배열 쌍 반환.
- 두 함수 모두 `tests.js`에 assert 테스트 추가.

## 배포

`sw.js`의 `CACHE` 버전 문자열을 올린다 (`eok-v7` → `eok-v8`).

## 제외 (YAGNI)

- 기본 카테고리 삭제·수정·숨김
- 카테고리 순서 변경, 색상/아이콘
- 카테고리 삭제 시 과거 기록 재분류 흐름
