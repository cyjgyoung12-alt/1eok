# 1억 모으기

모바일 앱처럼 쓰는 개인 자산 목표 대시보드입니다. "한 달에 5분, 빼먹어도 안 망가지는
현실 거울" + "오늘 이길 수 있는 게임"을 목표로 합니다.

- 오늘 쓸 수 있는 돈을 매일 보여주고, 지출/수입 업데이트를 최대한 간편하게 만들기
- 1억까지 남은 거리와 예상 달성 시점을 월 1회 결산으로 정직하게 재동기화하기

## 실행 방법

### 가장 간단한 방법

`index.html`을 브라우저로 열면 바로 실행됩니다.

### VSCode에서 로컬 서버로 실행

MacBook 기준:

```bash
python3 -m http.server 4173
```

그 다음 브라우저에서 엽니다.

```text
http://localhost:4173
```

Node/npm을 쓰고 싶다면:

```bash
npm run dev
```

### 테스트

`logic.js`(순수 계산 코어)에 대한 콘솔 기반 assert 테스트입니다. 의존성 없이 node로 실행합니다.

```bash
node tests.js
```

## 파일 구조

```text
index.html      앱 진입점
styles.css      Void 다크 테마 UI 스타일

tests.js        logic.js에 대한 콘솔 기반 assert 테스트

```

## 현재 구현된 핵심 기능

앱은 3개 레이어로 구성됩니다.

- **미션** (매일): "오늘 쓸 수 있는 돈"을 매일 재분배하는 예산. 이번 달 변동지출
  예산에서 지금까지 쓴 만큼을 빼고 남은 날수로 나눠 오늘 쓸 수 있는 금액을 다시
  계산한다. 초과해도 실패로 취급하지 않고 내일 예산이 줄어들 뿐이며, 기록이 있는
  날만 클리어/실패로 판정해 연속 클리어 스트릭을 보여준다
- **기록** (수시, 옵션): 지출·수입 입력이 미션 현황을 즉시 갱신한다. 월급·월세·통신비
  등 고정 항목은 매월 자동으로 거래에 반영된다
- **결산** (월 1회): 계좌별 잔고를 입력하면 순자산이 현실 값으로 재동기화되고, 즉시
  월간 운용 보고서(자산 증감, 계좌별 변화, 저축률, 도착 예상일 변화)가 생성된다.
  결산 이력은 최근 12개월 롤링 그리드로 첫 화면에 요약된다

속도 계산은 최근 결산 델타(최대 3개 평균, 갭 월은 월평균 정규화)를 우선 사용하고,
결산이 1건뿐이면 필요 월저축을 "목표 페이스 가정"으로 명시해 사용한다.

데이터 저장은 브라우저 `localStorage`, 백업/복원은 JSON 파일로 지원한다.

## 중요한 제품 방향

이 앱은 일반 가계부가 아닙니다. "한 달에 5분, 빼먹어도 안 망가지는 현실 거울" +
"오늘 이길 수 있는 게임"이 목표입니다. 결산은 보상 장치가 아니라 데이터 공급·
재동기화 장치이며, 기록 오차가 쌓여도 매달 결산이 진실을 복원하므로 기록은 부담
없는 옵션으로 유지되어야 합니다.

보상은 정직해야 합니다. 돈이 아니라 행동(연속 기록, 연속 결산)만 칭찬하며, "목표에
다가갔습니다" 류의 과장된 문구는 쓰지 않습니다.

첫 화면(현실 탭)은 계속 단순해야 합니다.

1. 미션 카드(오늘 쓸 수 있는 돈·잔여·클리어 스트릭)
2. `− 지출` / `+ 수입` 버튼
3. 1억 카드(남은 금액·현재 자산·도착 예상일·계산 기준 라벨)
4. 결산 카드(결산 D-day·롤링 12개월 그리드·연속 결산 수)

월간 보고서 상세, 카테고리별 지출, 계좌 관리, 고정비, 백업, 계산 기준 설명은 흐름
탭·설정 탭·시트로 보냅니다.
