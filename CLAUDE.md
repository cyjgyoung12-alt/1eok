# Claude Code 작업 지침

이 프로젝트는 “1억 모으기”를 위한 모바일 앱형 정적 웹앱입니다.

## 최우선 제품 원칙

항상 이 두 가지를 우선합니다.

1. 사용자가 현재 목표 대비 어떤 현실인지 즉시 느끼게 한다.
2. 수입과 지출 업데이트를 매우 쉽고 빠르게 만든다.

기능을 늘릴 때 첫 화면이 복잡해지면 안 됩니다. 첫 화면은 “현실 확인 + 빠른 입력”만 담당합니다.

## 첫 화면 유지 원칙

첫 화면에는 아래 정보만 유지합니다.

- 1억까지 남은 금액
- 현재 자산
- 지금 속도로 하면 언제 달성 가능한지
- 계산 기준 라벨
- `+ 지출`
- `+ 수입`
- 이번달 수입
- 이번달 지출

다음 정보는 첫 화면에 직접 노출하지 말고 팝업, 흐름 탭, 설정 탭으로 보냅니다.

- 고정비 상세
- 카테고리별 지출
- 변동자산 상세
- 백업/복원
- 계산식 설명
- 최근 3개월 추세

## 금액 정의

앱의 목표는 “투자 가능 순자산 1억”입니다.

포함:

- 현금
- 예금/적금
- CMA/파킹통장
- 주식/ETF/펀드 평가금액
- 코인 평가금액

주의:

- 주식/코인 평가금액은 수입이나 지출이 아니라 변동자산 평가 변화로 반영합니다.
- 실거주 부동산, 자동차, 중고 물건 등은 메인 1억 목표에서 제외하는 방향을 유지합니다.

## 속도 계산 원칙

목표 달성 시점은 `이번 달 저축액`만으로 계산하지 않습니다. 월중에는 계속 변하기 때문입니다.

우선순위:

1. 최근 완료된 3개월 기록이 충분하면 최근 3개월 평균 저축액
2. 데이터가 부족하면 설정된 월 목표 저축액
3. 월 목표 저축액도 없으면 속도 부족

## 입력 UX 원칙

지출/수입 입력은 최대한 가벼워야 합니다.

- 금액 입력이 첫 번째
- 카테고리는 칩으로 빠르게 선택
- 메모와 날짜는 접힌 옵션
- 저장 후 바로 닫기
- 저장 후 토스트로 결과 확인

## 변동자산 원칙

변동자산은 `흐름` 탭에서 관리합니다.

- MTS/코인 자산 화면 캡쳐 업로드
- 자동 금액 후보 추출
- 사용자가 최종 금액 확인
- 변동자산 평가액으로 반영
- 이력 저장

OCR은 오인식될 수 있으므로 반드시 사용자가 확인한 금액만 저장합니다.

## 기술 구조

현재는 빌드 없는 정적 웹앱입니다.

- `index.html`
- `styles.css`
- `app.js`
- `manifest.json`

외부 의존성을 추가하지 않는 것을 기본으로 합니다. 프레임워크 도입은 사용자가 명시적으로 원할 때만 검토합니다.

## 스타일 방향

- 모바일 앱 기준
- 카드 반경은 8px 이하 유지
- 첫 화면 정보 밀도는 낮게 유지
- 버튼은 명확하고 크게
- 텍스트가 버튼/카드 안에서 넘치지 않게
- 과한 장식, 랜딩페이지식 표현, 설명 문구 남발 금지
