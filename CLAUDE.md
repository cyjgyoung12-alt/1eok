# Claude Code 작업 지침

이 프로젝트는 “1억 모으기”를 위한 모바일 앱형 정적 웹앱입니다.

## 최우선 제품 원칙

포지셔닝: **“한 달에 5분, 빼먹어도 안 망가지는 현실 거울”** + **“오늘 이길 수 있는 게임”**.

앱은 3개 레이어로 구성됩니다.

| 레이어 | 주기 | 역할 | 실패 허용 |
|---|---|---|---|
| 미션 | 매일 | “오늘 쓸 수 있는 돈” 재분배형 예산. 매일 여는 이유 | 초과해도 실패 없음, 내일 예산이 줄어들 뿐 |
| 기록 | 수시(옵션) | 지출/수입 입력 = 미션 현황 갱신. 고정비는 자동 기록 | 빈 날은 판정 없음, 스트릭 안 끊김 |
| 결산 | 월 1회 | 계좌별 잔고 입력 → 자산을 현실로 재동기화 → 월간 보고서 | 한 달을 빼먹어도 다음 결산이 진실을 복원 |

결산은 보상 장치가 아니라 데이터 공급 + 재동기화 장치입니다. 기록이 며칠 빠져도 다음
결산이 오차를 교정하므로 기록은 부담 없는 옵션이어야 합니다.

**정직한 보상만**: 돈이 아니라 행동을 칭찬합니다. 지출/수입 입력에 “목표에
다가갔습니다” 같은 거짓 칭찬 문구를 넣지 않습니다. “이번 달 N일째 기록”처럼 행위
연속성만 보상합니다. 보상 주기는 실제 숫자가 움직이는 주기와 일치시킵니다(일=미션,
월=결산·보고서).

기능을 늘릴 때 첫 화면이 복잡해지면 안 됩니다. 첫 화면은 “오늘의 미션 + 현실 확인 +
빠른 입력”만 담당합니다.

## 첫 화면 유지 원칙

첫 화면(현실 탭)은 위에서 아래로 아래 4개 카드/버튼만 유지합니다.

1. **미션 카드**: 오늘 쓸 수 있는 돈, 오늘 잔여, 클리어 스트릭
2. `− 지출` / `+ 수입` 버튼
3. **1억 카드**: 1억까지 남은 금액, 현재 자산, 도착 예상일, 계산 기준 라벨
4. **결산 카드**: 이번 달 결산 D-day, 최근 12개월 롤링 그리드, 연속 결산 수

다음 정보는 첫 화면에 직접 노출하지 말고 시트(팝업), 흐름 탭, 설정 탭으로 보냅니다.

- 월간 보고서 상세(자산 증감, 계좌별 변화, 저축률)
- 카테고리별 지출
- 계좌 관리(추가·이름변경·비활성·순서)
- 고정비 상세
- 백업/복원
- 계산식 설명(계산 기준 시트)

## 금액 정의

앱의 목표는 “투자 가능 순자산 1억”입니다.

포함:

- 현금
- 예금/적금
- CMA/파킹통장
- 주식/ETF/펀드 평가금액
- 코인 평가금액

주의:

- 주식/코인 평가금액은 수입이나 지출이 아니라 월말 결산의 계좌 잔고로 반영합니다(증권·코인 계좌 포함).
- 실거주 부동산, 자동차, 중고 물건 등은 메인 1억 목표에서 제외하는 방향을 유지합니다.

## 속도 계산 원칙

목표 달성 시점은 `이번 달 저축액`만으로 계산하지 않습니다. 월중에는 계속 변하기 때문입니다.

우선순위:

1. 최근 결산 델타 최대 3개의 평균(갭이 있는 달은 개월 수로 나눠 월평균으로 정규화)
2. 델타가 없으면(결산이 1건뿐이면) 필요 월저축을 가정값으로 사용하되, 화면에는
   반드시 “목표 페이스 가정” 라벨을 명시해 실제 속도가 아님을 드러낸다
3. 결산 자체가 없으면 속도 부족(온보딩이 첫 결산을 강제하므로 정상 상태에서는 발생하지 않음)

## 입력 UX 원칙

지출/수입 입력은 최대한 가벼워야 합니다.

- 금액 입력이 첫 번째
- 카테고리는 칩으로 빠르게 선택
- 메모와 날짜는 접힌 옵션
- 저장 후 바로 닫기
- 저장 후 토스트로 결과 확인
- 지출 저장 토스트는 오늘 잔여 예산을 보여준다

## 결산 원칙

결산은 `현실` 탭의 결산 카드 → 결산 시트에서 월 1회 진행합니다.

- 계좌별 잔고를 입력받는다(직전 결산 값을 프리필). 같은 달에 다시 결산하면 새 값이
  그 달 결산을 덮어쓴다(교체, 누적 아님)
- 결산 시점의 `requiredSaving`, `monthlyIncome`은 스냅샷으로 함께 저장한다. 이후
  목표 금액/목표일/월 수입을 바꿔도 과거 결산의 스냅샷과 그리드·보고서 판정은
  소급 변경되지 않는다
- 결산이 순자산의 진실원(source of truth)이다. 결산과 결산 사이의 거래 기록(지출/수입)은
  그 사이 자산 변화를 추정하는 보정치일 뿐이며, 다음 결산이 오차를 항상 교정한다

OCR·이미지 업로드 기반 자동 인식은 사용하지 않습니다. 계좌 잔고는 사용자가 직접 입력합니다.

## 기술 구조

현재는 빌드 없는 정적 웹앱입니다.

- `index.html`
- `styles.css`
- `app.js` — 상태, 화면 렌더링, 이벤트 처리
- `logic.js` — 순수 계산 코어(예산·판정·속도·도착일 등). DOM 의존 없음, node 테스트 대상
- `tests.js` — `logic.js`에 대한 콘솔 기반 assert 테스트(의존성 없음)
- `sw.js` — 서비스워커(PWA 앱셸 캐시-우선)
- `manifest.json`
- `icons/` — PWA/홈 화면 아이콘

localStorage 키는 `one_hundred_million_mobile_v2`입니다(v1과 마이그레이션 없음).

테스트: `node tests.js`

외부 의존성을 추가하지 않는 것을 기본으로 합니다. 프레임워크 도입은 사용자가 명시적으로 원할 때만 검토합니다.

## 스타일 방향

Void 테마 — 다크 고정, 라이트 테마 없음(`color-scheme: dark`).

- 배경 `#0b0d10` / 카드 `#12151a` / 라인 `#22262e` / 텍스트 `#f2f4f7` / 뮤트 `#8b93a1`
- 액센트는 민트 `#3ce8a4` 하나만 사용. 지출 강조는 색이 아니라 부호·보더로 구분
- 숫자는 `font-variant-numeric: tabular-nums`로 정렬
- 카드 반경은 8px 이하 유지
- 첫 화면 정보 밀도는 낮게 유지
- 버튼은 명확하고 크게
- 텍스트가 버튼/카드 안에서 넘치지 않게
- 글로우·그림자 과다·이모지·과장 문구 금지. 문구는 분석가 톤(예: "저축률 34.2% · 목표 대비 +2.1%p")

## 실행

MacBook:

```bash
python3 -m http.server 4173
```

또는:

```bash
npm run dev
```

브라우저:

```text
http://localhost:4173
```

테스트:

```bash
node tests.js
```

서비스워커는 앱셸을 캐시-우선으로 서빙합니다. 배포 시 `sw.js`의 `CACHE` 버전 문자열을 올려야 클라이언트가 새 배포를 받습니다.
